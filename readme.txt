2.4. Тестирование сотрудников 

 

Задание 
	Данные хранятся в реляционной БД: 
	1. Для сотрудников (справочник): ФИО, логин, пароль (не в открытом виде).  

2. Роль сотрудника (справочник): администратор (может добавлять тесты), пользователь (нет возможности добавлять тесты). 
	3. Для тестов: тип теста (справочник), вопрос, варианты ответов. критерии оценки. 
	4. Для результатов тестов: сотрудник - тест - результат (включая выбранные пользователем варианты ответов). 
 
	Пользовательский интерфейс должен обеспечивать следующие возможности: 
	1. Регистрация/авторизация пользователя по логину/ФИО (выбор из справочника с возможностью поиска) и паролю. В случае регистрации пользователя предусмотреть проверку по логину и/или ФИО.  

2. Добавление оценки (для администраторов): неудовлетворительно, удовлетворительно, хорошо, отлично.  

3. Добавление типа теста (для администраторов): одно значение результата, множественный выбор результата. 
	4. Добавление теста (для администраторов): выбор типа теста (из справочника), варианты ответа, для оценки (выбирается из справочника) указывается диапазон процентного соотношения правильных ответов.  

5. После авторизации пользователь имеет возможность пройти тест в случае, если он не проходил его раньше. 
	6. Варианты ответа на тест должны перемешиваться случайным образом.  

7. После ответа на вопрос пользователем осуществляется переход на следующий вопрос без отображения правильного варианта ответа.  

8. По окончании тестирования отображается процент правильных ответов на вопрос и оценка. Вопрос с множественными вариантами ответов считается правильным в случае, если выбраны все правильные варианты ответа. 
	9. Для администраторов: выбор теста, для которого отображать результаты ответов на вопросы (ФИО, процент правильных ответов, оценка). Результаты должны отображаться на экране, а также должна быть возможность выгрузки в Excel. При щелчке мышкой на ФИО отображать выбранные варианты ответов по каждому вопросу. Отображать зеленым цветом верные ответы, красным ошибочные.
